# Group World —— 让你的群聊活成一个世界

> **群聊叙事操作系统。** 不是 speaker selector，不是发言过滤器——它是一个完整的 Agent 运行时，让你的群聊变成一个持续演化的世界。

---

## 它能做什么

**让群聊有导演。** 不会所有人一起抢话了。Director 每轮决定谁发言、按什么顺序、每人收到什么舞台指令（其他角色都看不到）。场面失控的感觉消失了。

**角色会记住你半年前说过的话。** Memory 系统自动从对话中提取关键事件，跨对话持久化。下次聊到相关话题，角色会自己想起来。

**剧情不会烂尾。** 导演账本持续记录故事进度、角色关系、阵营变化。八十轮之后导演还知道故事在第几章、谁和谁有旧账。长线叙事第一次有了可维护的状态。

**世界书不再吃灰。** 不再依赖 ST 的关键词碰运气。Director 会主动扫描世界书条目，把该注入的信息写进角色的上下文。

**NPC 自己长出来。** 故事需要路人、店主、卫兵？NPC 系统在需要时自动生成，带背景、性格、对话风格。不需要提前手写角色卡。

**AI 会自我检查。** Critique 系统在导演决策后自动审视剧本质量——角色抢戏了吗？剧情跑偏了吗？然后修正。

**配置一键切换。** 不同的世界观、不同的导演风格、不同的角色设定——存成配置档，点一下切换。导出分享给别人，零门槛。

**内置万能助手：暮羽。** 一只住在插件里的猫头鹰娘，带着全套 Group World API 文档。不知道某个功能怎么用？问她。配置跑偏了不知道怎么调？问她。想写扩展不知道从哪下手？问她。

---

## 不止如此：可扩展的 Agent 运行时

上面这些是开箱就有的。但 Group World 的底层是一个完整的 **Agent Runtime + Provider 扩展框架 + Prompt DSL**。

**如果你会写一点东西：**
- **Custom Prompt** — 把大段世界观设定收进一个占位符，Prompt 干净可维护
- **Custom Agent** — 自定义 LLM 驱动的子任务，加入 Director 管线
- **Capability 扩展** — 给角色加 TTS、情绪分析、图片生成能力

**如果你会写 JS：**
- **Provider 注册协议** — 自定义数据源，注入到运行时任意 Prompt 位置，DSL 路径查询直达字段
- **Script Executor** — 在导演决策、角色发言、回合结束三个时机插入你的代码，修改决策、读写共享状态
- **完整 Agent 框架** — 注册任意数量的自定义 Agent，自定义管线顺序、上下文访问权限、解析校验逻辑

内置的东西都是这套框架上的实例。你的扩展和内置的逻辑跑在同一个运行时上。

---

## 打破第四面墙

角色能知道的，不限于你写在角色卡里的东西。你可以让实时天气影响角色情绪，让股价波动触发剧情事件，让角色读取外部的规则书、设定集、甚至另一个 AI 的输出。

角色能做的，不限于 ST 内置的能力。你可以让角色真的说出语音，在关键时刻自动生成插图，让外部程序响应角色的某个决策。

一切都可以搬进搬出。角色、世界书、配置档、档案、记忆、NPC、总结、批判——每种数据都有独立的导出/导入通道。把跑了半年的群聊状态完整打包，换台机器继续。

**这不是"群聊里有个机器人"。这是叙事世界和真实世界之间的一道可控边界。**

---

## 不是什么

- **不是发言过滤器。** 不是在 ST 输出之后做筛选——而是在角色生成之前增加一层决策。角色还没开口，Director 已经在调度了。
- **不是提前设计好的世界。** 状态结构不由开发者在代码里定义。Director 在运行过程中自然生成状态，Ledger 记录，Provider 发布，DSL 查询，Agent 消费。给机制，不给设定。
- **不是一个"群聊增强工具"。** 它是一套让群聊变成自演化叙事世界的基础设施。

## 快速上手

1. **安装** — ST 扩展面板 → 安装新拓展 → 输入 `https://github.com/Windy-Sora/SillyTavern-GroupWorld` → 启用
2. **应用默认配置** — 设置面板 → 仪表盘 → 配置档下拉框选 `group-world-default` → 应用
3. **生成角色档案** — 点击仪表盘「生成档案」按钮
4. **勾选世界书** — 点击仪表盘世界书统计方块，勾选你想启用的条目
5. **进入群聊，开始。**
6. **🦉 强烈建议点一下「召唤暮羽」按钮。** 她会自动导入到你的角色列表，带着全套 API 文档，任何问题直接问她。

## ⚠️ 高权限操作声明

以下声明遵循类脑社区《恶意代码防范规定》中关于高权限/敏感操作的强制声明要求。

### 卡外数据操作

| 类型 | 操作对象 | 用途 |
|------|----------|------|
| 读取 | 所有角色卡数据（名称、描述、头像、性格、场景设定） | 导演调度需了解群内角色信息；NPC 生成需检查名字冲突；角色档案 Agent 需构建角色画像 |
| 读取 | 已激活的世界书条目 | 世界书扫描器主动将相关条目注入角色上下文 |
| 写入 | 角色卡（通过 ST API 创建） | NPC 系统自动生成路人角色；暮羽助手导入内置帮助角色 |
| 写入 | 世界书（通过 ST API 导入/激活） | 暮羽助手导入配套助手世界书 |

### 卡内数据操作

| 类型 | 操作对象 | 用途 |
|------|----------|------|
| 读取 | 聊天记录（当前群聊的完整对话历史） | 记忆提取、上下文总结、批判审视、导演决策——所有 Agent 都需阅读对话历史 |
| 写入 | `chat_metadata`（聊天元数据） | 持久化叙事状态：NPC 列表、角色记忆、导演账本、批判结果、自动任务计数器、摘要 |
| 写入 | Extension Prompt（角色提示词） | Director 通过 extension prompt 向每个发言角色传递私有舞台指令，其他角色不可见 |
| 写入 | 角色切换（setCharacterId / setCharacterName） | 强制发言和发言后处理 Agent 在处理时需要临时切换当前角色 |

### 代码执行

| 操作 | 说明 |
|------|------|
| Provider / Capability 导入 | 用户可通过文件选择器导入任意 `.js` 文件，通过 `import()` 动态加载，在页面上下文运行，拥有与插件同等权限。导入前插件会对源码进行静态扫描，检测到危险 API（`fetch()`、`eval()`、`document.cookie`、`localStorage` 等）时弹窗警告，但用户仍可选择继续导入。**请仅导入绝对信任来源的代码。** |
| Script Executor | 用户可在设置面板编写自定义 JS 代码，在三个时机自动执行：导演决策后、角色发言前、回合结束后。代码在页面上下文运行，可通过 `getContext()` 访问 ST API，可通过 `turnShared` 读写共享状态。**请仅运行自己编写或绝对信任的代码。** |

### 设备信息读取

| 操作 | 用途 |
|------|------|
| 读取系统时间（`new Date()` / `Date.now()`） | `{{systemTime}}`、`{{timeOfDay}}`、`{{moonPhase}}` 三个 Provider 读取设备系统时钟，用于叙事上下文注入（当前日期时间、季节时段、月相），不离开浏览器，不上传任何服务器 |
| `{{randomDice}}` Provider | 使用 `Math.random()` 生成随机数，不涉及网络请求 |

### API Key 存储

用户为各 Agent 配置的 API Key 存储在 ST 本地 `extension_settings` 中，仅由浏览器端发出 LLM 请求时使用，不经过作者或任何第三方服务器。

## 🌐 第三方请求披露

本插件支持用户为各 Agent（导演、记忆、批判等）自定义 LLM API 端点。用户可在设置面板中配置任意 OpenAI-compatible（`/v1/chat/completions`、`/v1/models`）或 Anthropic-compatible（`/v1/messages`）的 API 地址。

**所有 LLM 请求的目标地址完全由用户配置，由浏览器直接发出，数据不经过作者服务器。** 用户自行对其配置的 API 端点的隐私政策和数据安全负责。

插件自身不向任何外部域名发起请求。所有请求目标为 ST 本地服务器（`localhost`）或用户自行配置的 API 端点。
