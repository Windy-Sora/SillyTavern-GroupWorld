## 1. 快速开始（5 分钟上手）

### 安装

将 `SillyTavern-GroupWorld` 文件夹放入 ST 的 `public/scripts/extensions/third-party/` 目录，重启 ST。在扩展管理面板中启用。

### 三步启动

**第一步：选模式** → 打开「导演」抽屉，选 LLM 模式（效果更好）或公式模式（零 token）。

**第二步：准备角色** → 打开「角色」抽屉，展开「角色档案」卡片 → 全部重新生成。展开「角色记忆」卡片 → 提取全部。

**第三步：开始对话** → 进入群聊，正常发送消息。角色不再抢话，按导演顺序发言。

### 更快的方式：领养暮羽

仪表盘底部点击 **「🦉 领养暮羽」**——GD 插件的内置开发助手，一只住在紫色台灯下的猫头鹰娘。一键导入角色卡 + 世界书，然后直接和她对话：

- **不知道怎么配？** 问她。世界书里有完整的配置指南。
- **想写 Provider / Capability / Custom Agent / Script Executor？** 问她。她会按决策流程帮你找到最优雅的方案。
- **出 bug 了？** 问她。她知道查哪些日志、看哪些计数器。

暮羽不是代码生成器，她是 GD 的首席架构师。她不会堆功能，而是引导你用现有机制组合解决问题。

### 更快的方式：预设

仪表盘底部预设下拉框中选择 `group-director-default` → 点击「应用」——所有推荐配置即刻生效。

### 界面分层说明

| 级别 | 区域 | 什么时候用 |
|------|------|------|
| **一眼可知** | 仪表盘 | 每次打开——当前模式、上次决策、数据统计、预设切换 |
| **核心配置** | 导演 / 角色 | 第一天——选模式、写 Prompt、生成档案记忆 |
| **调优** | 连续性 | 第一周——管理账本、世界书、上下文总结 |
| **按需** | 反应 / 工具 | 按需开启——多模态、导出导入、Agent 配置、自定义 Agent、调试 |

> **建议学习路径**：点击「领养暮羽」→ 和她聊聊你的需求 → 她会一步步教你怎么配。也可以先仪表盘了解状态 → 导演抽屉配模式 → 角色抽屉准备数据 → 开始对话。高级抽屉默认折叠，不干扰日常使用。

## 2. 仪表盘 — 导控台

打开 Group World 设置面板后，第一条内容就是仪表盘。它**不属于任何抽屉，始终可见**。

### 状态指示灯

最左侧的彩色圆点：

| 指示灯 | 含义 |
|------|------|
| 🟢 绿色呼吸灯 | LLM 模式，导演正在工作 |
| 🟡 琥珀色 | 公式模式，本地评分中 |
| ⚫ 灰色 | 已关闭 |

右侧显示模式名称和语言切换。

### 上次决策

指示灯下方一行带左边框的摘要，显示导演上轮的选择：

> **LAST CALL** Alice → Bob → Charlie
> *Alice directly addressed Bob and Charlie has relevant intel*

### 数据统计

七个数值方块 + 手动刷新按钮。打开设置面板时自动刷新。**所有方块均可点击展开**，显示详细列表。

| 统计 | 点击展开内容 |
|------|-------------|
| 档案 | 角色名 + 状态（绿=就绪/橙=生成中/红=失败）；展开行查看 summary/tags/motivation |
| 记忆 | 角色名 + 条目数；展开行查看最近 5 条记忆 |
| NPC | 角色名 + 描述摘要；展开行查看完整描述/性格/背景 |
| 账本 | 最近 8 轮发言顺序 + 理由；展开行查看完整理由和剧本 |
| 总结 | 显示 `已覆盖/总消息数`；展开行查看当前总结内容，支持编辑 |
| 世界书 | 全选/取消全选 + 逐个勾选 |
| 🔄 | 手动刷新所有数据（点击时图标旋转） |

#### 行内编辑

展开面板中悬停任意字段会浮现「编辑」按钮，点击后字段变为文本框，Ctrl+Enter 或点「保存」写入数据，点「取消」放弃。编辑直接写回底层存储，与抽屉中对应编辑区保持同步。

| 面板 | 可编辑字段 |
|------|-----------|
| 总结 | summary 内容 |
| 档案 | summary |
| 记忆 | event 文本 |
| NPC | description / personality / scenario |
| 账本 | reason |

数据也会在每次展开/折叠抽屉时自动刷新。总结面板和记忆面板底部各有一组自动触发控件（启用/自动/每N条触发），与对应抽屉卡片双向同步。

### 快捷操作

四个快捷按钮，仅在对应系统启用时显示：

| 按钮 | 效果 | 显示条件 |
|------|------|----------|
| 扫描存档 | 从聊天存档恢复档案 + 刷新记忆列表 | profile 或 memory 启用 |
| 生成档案 | 为群聊所有角色生成角色档案 | profile 启用 |
| 提取记忆 | 顺序为每个角色提取新记忆（不弹确认框） | memory 启用 |
| 执行总结 | 执行上下文总结。若未启用总结系统则自动开启 | 始终（群聊中） |

### 配置档管理

仪表盘底部一行：下拉框 + 应用 + 导入 + 刷新按钮。

下拉框中的选项以分组形式组织：

| 分组 | 内容 | 点击「应用」时 |
|------|------|---------------|
| **内置配置档** | `group-director-default` 等出厂预设 | 先加载到列表，再应用设置 |
| **我的配置档** | 用户保存或导入的配置档 | 直接应用设置 |

右侧按钮：

| 按钮 | 效果 |
|------|------|
| 应用 | 应用当前选中的配置档到全局设置（需刷新页面完全生效） |
| 导入配置档 | 选择 `.zip` 文件导入外部配置档 |
| 🔄 刷新 | 手动同步下拉框中的配置档列表（保存/删除后下拉框自动刷新） |

工具抽屉的配置档卡片中也有同款下拉框，两边选项始终保持同步。

## 3. 抽屉与卡片指南

### 阅读约定

- **抽屉**：点击标题展开/折叠的整块区域（ST 原生 `inline-drawer`）
- **卡片**：抽屉内部的可折叠信息块。每个卡片有标题栏 + 展开箭头，点击展开/收起。**卡片折叠状态会被记住**，下次打开设置时保持不变

所有控件更改**即时保存**，无需手动保存按钮。

---

### 3.1 导演

**谁发言、按什么顺序、怎么演。** 整个插件的核心抽屉。

#### 模式选择

三个并排按钮：**关闭** / **公式** / **LLM**。互斥单选。

#### 公式参数

仅公式模式可见。控制本地评分的具体参数——Top-N、评分权重、触发器、主动性。详见[公式判断](#公式判断零-token)章节。

#### LLM 参数

仅 LLM 模式可见。包含四张子卡片：

**Director Prompt**

| 控件 | 作用 |
|------|------|
| 每轮最多发言人数 | 导演每轮允许几人说话。推荐 2-3 |
| 上下文层数 | 给导演看最近 N 条消息。推荐 10 |
| Director Prompt 模板 | **核心配置**。使用占位符注入数据。模型必须返回 JSON |
| 恢复默认 | 重置为内置 Prompt |

**连贯性 & 账本**（展开卡片）

| 控件 | 作用 |
|------|------|
| 记录导演账本 | 每次决策后保存完整 JSON 到聊天元数据 |
| 清空 | 删除当前全部账本记录 |
| 使用导演历史 | 将历史账本注入当前 Prompt，保持剧情连续性 |
| 连贯剧本模式 | 仅上一轮 / 完整历史 |
| 历史轮数 | 完整历史模式下注入多少轮 |
| 包装模板 | 控制历史 JSON 如何包裹注入 |

**世界书 & 知识库**（展开卡片）

| 控件 | 作用 |
|------|------|
| 世界书注入 | 将激活的世界书条目注入导演 Prompt |
| 包装模板 | 控制世界书文本的包裹格式 |
| 知识库 | 存放不会被渲染的参考文本。通过 `{{knowledge}}` 引用 |

**强制发言**（展开卡片）

ST 群聊原生的强制发言功能需要插件适配处理，否则会导致时间线错乱。

| 选项 | 效果 |
|------|------|
| 原生放行 | 弹窗确认后让 ST 自行处理 |
| 直接截停 | 完全阻止强制发言 |
| LLM 接管 | 调用模型为该角色生成剧本并计入账本 |

**模板 & 角色描述**（展开卡片）

| 控件 | 作用 |
|------|------|
| 递归渲染 | 占位符输出中的 `{{...}}` 继续解析 |
| 最大递归轮数 | 防无限循环。推荐 5 |
| 调试占位符 | 保留未识别占位符方便排查 |
| 角色描述控制 | 全量传入 / 切片截断（推荐 200 字符） |

#### 导演剧本 (Director Script)

勾选「启用导演剧本」后，导演为每个发言角色输出舞台指令，注入角色 prompt 指导表演。

| 控件 | 作用 |
|------|------|
| 注入位置 | Prompt 开头（弱干预）/ 对话附近（强干预） |
| 剧本风格要求 | 告诉导演希望的剧本风格 |
| 角色 Prompt 注入模版 | **角色信息的唯一总注入口**。`{{script}}` 替换为实际剧本，`{{charMemoryCurrent}}`、`{{charCritique}}`、`{{characterLore}}` 等角色级 Provider 均在此渲染。可恢复默认 |

#### 高级：JSON 输出格式

Director 要求 LLM 返回 JSON。用户可自定义 JSON schema 来控制输出字段。

| 控件 | 作用 |
|------|------|
| JSON 输出格式 | 自定义 LLM 返回的 JSON schema 模板。默认包含 speakers / reason / loreAssignments |
| `{{scriptField}}` | schema 中的占位符——开启导演剧本时展开为 scripts 字段，关闭时自动消失。可在 schema 中增删任意字段（如 emotion、pacing） |
| 恢复默认 | 重置为内置 JSON schema |

### 3.2 角色

**导演知道的角色数据。** 四个折叠卡片，每个默认只显示标题和状态标签。

#### 角色档案（卡片）

生成结构化的角色档案（特质、动机、关系），注入 Director Prompt 作为身份参考。状态标签显示就绪数量。

| 控件 | 作用 |
|------|------|
| 启用角色档案 | 总开关 |
| 扫描存档 | 从聊天存档恢复已有档案 |
| 检测变动 | 检测角色加入/退出 |
| Token 预算 | 超过时压缩非活跃角色 |
| 并发数 | 同时生成的角色数 |
| 生成 Prompt / Schema / 渲染模板 | 三个模板均可自定义并恢复默认 |
| 全部重新生成 | 一键为所有角色生成档案 |
| 档案管理卡片 | 每个角色可展开，支持编辑/重生成/删除 |

#### 角色记忆（卡片）

从对话历史中提取角色的关键经历和情感变化。状态标签显示条目总数。

| 控件 | 作用 |
|------|------|
| 启用角色记忆 | 总开关 |
| Token 预算 | 限制记忆文本总长度 |
| 提取/渲染/压缩 Prompt + Schema | 可自定义并恢复默认 |
| 压缩保留最近 | 压缩时保留最近 N 条原文 |
| 单角色上限 | 超出自动裁剪最旧条目 |
| 仅发言角色 | 自动提取时只处理导演账本中近期发言的角色，去重后避免全量调用 |
| 记忆管理按钮 | 刷新/扫描/提取/检测失联/重置 |
| 记忆卡片 | 每个角色可展开浏览/编辑/删除单条记忆 |

#### NPC 生成（卡片）

根据对话上下文批量生成 NPC，可导入为角色卡。状态标签显示 NPC 数量。

#### 身份锚定（卡片）

自定义提示词，统一声明档案/记忆/摘要的使用规则和优先级。通过 `{{identity}}` 手动注入到 Director Prompt 或 角色 Prompt 注入模版。不会自动注入。

### 3.3 连续性

**导演记得什么。** 三个折叠卡片。

#### 上下文总结（卡片）

将聊天上下文压缩为简洁摘要，降低 token 消耗。启用后 `{{chatSummary}}` 有内容。

| 控件 | 作用 |
|------|------|
| 启用 / 复用上次 | 基础开关 |
| 自动总结 / 每 N 条触发 | 达到消息数阈值自动执行总结 |
| 总结 Prompt | 可自定义并恢复默认 |

| 操作 | 按钮 |
|------|------|
| 扫描存档总结 / 清除禁用 / 隐藏禁用 | 管理已有总结 |
| 执行 / 重新 / 回退 / 重置 | 总结生命周期 |
| 编辑总结结果 | 直接编辑文本后保存 |

仪表盘总结面板中也有一组相同的自动总结控件，两边保持同步。总结内容在面板中支持行内编辑，修改后双向同步到抽屉的文本框。

#### AI 批判（卡片）

让 AI 回顾最近对话，批判导演决策质量和角色表现。启用后 `{{directorCritique}}`、`{{characterCritique}}`、`{{charCritique}}` 有内容。

| 设置项 | 说明 |
|--------|------|
| AI 批判 / 复用 | 启用批判系统，复用上次已覆盖范围 |
| 自动批判 / 每 N 条触发 | 达到消息数阈值自动执行批判 |
| 批判 Prompt | 可自定义系统提示词，留空使用内置默认 |
| JSON Schema | 自定义输出 JSON 字段结构，留空使用内置默认 |
| 执行 / 重新 / 回退 / 重置 / 刷新 | 批判生成与生命周期管理 |

**{{charCritique}} 约定范式**：不管 JSON Schema 如何自定义，每个角色的批判必须是一个 object，其中 string 值渲染为 `[key] value`，array 值渲染为 `[key]` 后跟缩进列表。

仪表盘暂未集成批判面板（后续版本会加上）。

#### 导演账本（卡片）

浏览、编辑、清空导演历史决策。状态标签显示轮数。Raw 模式切换 JSON/易读格式。生成中自动锁死。

#### 世界书（卡片）

勾选哪些 ST 世界书条目注入到导演 Prompt。刷新按钮同步变更。最大注入条目限制（推荐 15-30）。

### 3.4 反应

**角色发言后触发什么。** 默认抽屉折叠。

#### 消息级反馈（卡片）

每个角色说完话后调用 PostSpeech Agent 分析情绪、触发 TTS 等。⚠ 启用后每次角色发言增加 1 次 LLM 调用。

#### 回合级反馈（卡片）

一轮群聊全部结束后调用一次。适合生成场景配图、更新世界观等批量处理。⚠ 启用后每回合结束增加 1 次 LLM 调用。

#### 已注册能力（卡片）

启用/禁用各多模态能力（emotion / tts / image）。可通过资产管理导入自定义能力。

#### 决策记录（卡片）

最近的多模态策略决策。随聊天持久化，自动去重。

### 3.5 工具

**配置档、导出导入、Agent、自定义 Agent、自定义 Prompt、接口参考、脚本执行器、调试。** 默认抽屉折叠。九个折叠卡片。

#### 配置档管理（卡片）

保存、导出、导入整套插件配置为 `.zip`。可创建多套配置档按需切换。导出时可选范围（按抽屉勾选）。API 密钥自动脱敏。

#### 导出 / 导入（卡片）

四类数据的导出导入：群聊（角色卡+世界书 `.zip`）、档案（`.json`）、记忆（`.json`）、NPC（`.json`）、摘要（`.json`）。导入时自动匹配角色，同名弹窗确认。

#### Agent 配置（卡片）

每个 Agent（Director / ForceSpeak / Profile / Summary / NPC / Memory / PostSpeech）可独立设置 API 端点和密钥。支持 OpenAI 或 Anthropic 协议。可测试连接。

#### 自定义 Agent（卡片）

用户自定义 LLM Agent，每 N 条消息自动触发或手动执行。Agent 不需要编辑器的 GUI、按钮、生命周期——只需要写一段 Prompt 和可选的 JSON Schema，结果就会通过 `{{providerName}}` 占位符注入到任意 Prompt 中。所有实例共用工具抽屉「Agent 配置」中的 `custom-agent` API 配置。

| 控件 | 作用 |
|------|------|
| 名称 / providerName | 显示名 + 占位符名（`{{providerName}}`） |
| 启用 / 自动 / 间隔 | 开关 + 每 N 条消息自动触发 |
| 顺序 | 每轮结束按数字升序串行执行 |
| Prompt | 自定义提示词（支持 `{{占位符}}` DSL 路径查询） |
| Schema | 可选 JSON Schema，LLM 输出自动校验+解析 |
| 结果 | 存储于聊天元数据，通过 `{{?providerName:field}}` 引用 |

点击卡片头部展开/收起配置面板。

#### 自定义 Prompt（卡片）

创建自己的 `{{placeholder}}` Provider。命名仅限 `a-z 0-9 _`。每个 Prompt 可编辑/开关/删除。支持批量导出导入。

#### 用户扩展（卡片）

导入自定义 `.js` Provider 或 Capability 文件。必须导出 `register(deps)`。导入后立即生效，重启自动恢复。

#### 接口参考（卡片）

纯参考文档，列出系统中所有 31 个已注册的 Provider 占位符及其说明（中英双语）。支持搜索过滤、添加/编辑/删除自定义条目、导出导入 JSON 文件、一键恢复默认。无功能副作用——仅帮助用户快速了解可在 Prompt 中使用的 `{{placeholder}}`。

#### 脚本执行器（卡片）

在导演生命周期的三个关键时刻（决策后/消息后/回合后）注入自定义 JS 脚本。通过 `ctx` 访问决策、消息、角色、共享状态。支持类型化参数、优先级排序、跨脚本共享状态、导入/导出（含版本号）。

详见 [脚本执行器](#8-脚本执行器)。

#### 调试（卡片）

- **模板测试器**：输入模板 + 可选 Locals JSON，测试渲染结果，不触发任何生成
- **执行追踪**：查看 Agent 各阶段耗时和输出。需开启调试日志

## 4. 两种导演模式

### 公式判断（零 token）

```
角色得分 = 名字被提及 × mentionWeight（默认 30）
         + 关键词匹配 × triggerScore（默认 40）
         + 近期未发言 × recencyWeight（默认 20）
         − 连续发言 × consecutivePenalty（默认 15）
         + Talkativeness × talkativenessWeight（默认 10）
         + 随机主动性（每轮 0 ~ initiativeBaseScore）
```

得分最高的 Top-N 个角色允许发言。适合快速测试和轻量日常。

### 大模型判断

每轮额外调用一次 LLM。导演分析上下文后返回 JSON：

```json
{
  "speakers": ["Alice", "Bob"],
  "reason": "Alice was directly addressed, Bob has relevant knowledge",
  "scripts": { "Alice": "Notices Bob's hesitation...", "Bob": "Recalls the clue from earlier..." }
}
```

`speakers` 顺序 = 发言顺序。`scripts` 是可选字段（需开启导演剧本）。

## 6. DSL 路径查询语法

### 基本查询

```
{{?name:field.subfield}}          → 取嵌套字段
{{?name:array[0]}}                → 取数组第 1 项
{{?name:array[-1]}}               → 取数组最后 1 项
{{?name:[key=value].field}}       → 按 key 精确查找
```

### 实战示例

```
{{?character_profiles:Alice.summary}}
  → "一位身经百战的王国骑士，恪守荣誉准则..."

{{?character_profiles:$character.motivation}}
  → 角色 Prompt 注入模版 中自动取当前角色的 motivation

{{?npcList:[name=老格里姆].description}}
  → 查找名为"老格里姆"的 NPC 的描述

{{?directorLedger:scripts.$character}}
  → 角色 Prompt 注入模版 中取导演为当前角色分配的剧本

{{?worldBooks:allEntries[comment=地理与空间].content}}
  → 查找注释为"地理与空间"的世界书条目内容
```

### 块循环

```
{{#provider:arrayPath}}
  对每个元素渲染此块。$it 为当前索引
{{/provider}}
```

示例：
```
{{#npcList:all}}
  - {{?npcList:all[$it].name}}: {{?npcList:all[$it].description}}
{{/npcList}}
```

## 8. 脚本执行器

在群聊导演生命周期的三个关键时刻注入自定义 JS 脚本。不是 Agent，不消耗 token。

**入口**：工具抽屉 → 脚本执行器卡片（反应抽屉下方）。

### 触发点

| 触发点 | 时机 | 执行方式 | 可用上下文 |
|--------|------|----------|-----------|
| 决策后 | Director 决策完成，角色生成前 | 阻塞（10s 超时） | `ctx.decision`（speakers/names/reason/scripts，可修改） |
| 消息后 | 每个角色发言渲染后 | 即发即忘（5s 超时） | `ctx.message`、`ctx.character`、`ctx.decisionSnapshot` |
| 回合后 | 本轮所有角色发言完毕 | 即发即忘，去重（5s 超时） | `ctx.decisionSnapshot` |
| 两者 | 消息后 + 回合后 | 各自模式 | 对应阶段字段 |
| 全部 | 三个触发点 | 各自模式 | 对应阶段字段 |

### 执行上下文 (ctx)

所有脚本通过 `ctx` 访问数据：

```js
ctx.params        // 脚本参数（类型化，可渲染）
ctx.shared        // 回合共享状态（跨脚本、跨阶段传递数据）
ctx.decision      // [仅 decision] 导演决策对象，live reference 可直接修改
ctx.decisionSnapshot  // [message/round] decision 阶段快照，只读
ctx.message       // [仅 message] 当前消息对象 { name, mes, ... }
ctx.character     // [仅 message] 当前角色对象
ctx.chat          // 完整聊天数组
ctx.characters    // 群聊角色列表
ctx.group         // 当前群聊对象
ctx.settings      // 插件设置
ctx.getContext    // ST context 函数
```

### 共享状态

脚本设置「返回模式」为「写入共享状态」后，返回的 object 会合并到 `ctx.shared`，供同轮后续脚本读取：

```js
// 脚本 A (priority: 1, decision 触发)
return { x: 42 };

// 脚本 B (priority: 2, decision 触发)
console.log(ctx.shared.x); // 42
```

共享状态在每轮开始时自动重置，不持久化。

### 参数

每个脚本可定义类型化参数（string / number / boolean）。参数值通过 `ctx.params.key` 访问。开启「渲染参数」后，字符串类型参数的默认值会过一遍模板渲染（单次，不递归）。

### 执行顺序

同触发点的脚本按**优先级（priority）升序**执行。优先级可为负数。超时或出错的脚本被跳过，不影响后续脚本。

### 导入/导出

脚本可导出为 `.json` 文件（含版本号），支持跨群聊复用。导入时同名脚本弹窗确认覆盖。配置档管理同步包含脚本执行器。

### 快速示例

**打印决策信息**（decision 触发，调试用）：
```js
console.log('speakers:', ctx.decision?.speakers);
console.log('reason:', ctx.decision?.reason?.slice(0, 100));
```

**修改发言顺序**（decision 触发，交换首尾）：
```js
const sp = ctx.decision.speakers;
if (sp.length > 1) {
    [sp[0], sp[sp.length - 1]] = [sp[sp.length - 1], sp[0]];
}
```

**记录消息统计**（message 触发）：
```js
console.log(`${ctx.character?.name}: ${ctx.message?.mes?.length} chars`);
```

## 9. 常见场景配方

### 奇幻 RPG 长篇叙事

```
模式：大模型判断 → 每轮发言 2-3 → 角色描述切片 200
启用导演剧本 ✓ → 剧本要求「史诗奇幻，注重角色内心」
启用连贯性 → 完整历史 10 轮
启用档案 ✓ → 启用记忆 ✓ → 世界书注入 ✓
完成后：工具 → 配置档管理 → 保存为配置档
```

### 轻量日常聊天

```
模式：公式判断 → Top-N 2 → 提及权重 30 → 近期未发言 20 → 连续惩罚 15
关闭 LLM 导演、剧本、档案、记忆
```

### 悬疑推理

```
模式：大模型判断 → 每轮 2
Director Prompt 追加「保持悬疑，每段发言都可能是线索」
剧本要求「保持神秘，不揭示真相」
启用记忆 ✓（追踪角色各自知道什么信息）
```

### 快速测试角色

```
公式判断 → Top-N 1
角色抽屉 → 全部重新生成 → 提取全部记忆 → 勾选世界书 → 开始
```

## 13. AI 批判

批判系统从导演和角色两个维度评价每轮对话质量，通过三个 Provider 注入到 Prompt 中。

### JSON Schema 约定

批判 JSON 必须遵循两层结构：
- `directorCritique`: object — 导演维度的评价，每个字段渲染为 `[字段名] 内容`
- `characterCritiques`: map<string, object> — key 为角色名，value 为该角色的评价

默认 Schema：
```json
{
  "directorCritique": {
    "pacing": "节奏评价",
    "spotlight": "焦点分配评价",
    "suggestions": ["建议1", "建议2"]
  },
  "characterCritiques": {
    "角色名": {
      "consistency": "一致性评价",
      "interaction": "互动表现评价",
      "suggestions": ["建议1"]
    }
  }
}
```

用户可在 JSON Schema 文本框中自定义字段名和结构，省去默认字段、新增维度均可。`{{charCritique}}` 和 `{{directorCritique}}` 完全动态渲染，适配任意 Schema。

### DSL 查询示例

```
{{?characterCritique:Alice.consistency}}          → Alice 的一致性
{{?characterCritique:Bob.suggestions[0]}}         → Bob 的第一条建议
{{#characterCritique:all}}                         → 遍历所有角色
  {{?characterCritique:all[$it].name}}
{{/characterCritique}}

{{charCritique}}                                   → 自动解析为当前角色的批判
```

## 14. 常见问题

**Q: 导演不工作了？**
A: 检查仪表盘指示灯不是灰色。LLM 模式检查工具 → Agent 配置 → Director 的 API 连接测试。

**Q: 角色档案为空？**
A: 需要先生成——角色抽屉 → 展开角色档案卡片 → 全部重新生成。

**Q: API 密钥会泄露吗？**
A: 导出配置档时 Agent API 密钥自动清空。导入后需重新填写。

**Q: 自定义 Prompt 名字被拒绝？**
A: 仅限 `a-z 0-9 _`。不能与内置 Provider（`recentMessages` 等）或 ST 宏（`user` `char` `time` 等 72 个）重名。

**Q: 怎么彻底还原配置？**
A: 仪表盘预设下拉框中选择 `group-director-default` → 点击应用。

**Q: 记忆压缩和记忆导出的关系？**
A: 压缩用 LLM 将旧记忆合并为摘要。压缩记忆带标记，导出时可选跳过。

**Q: 设置项太多了，找不到某个配置？**
A: 从仪表盘的数据统计入手——档案数量不对 → 角色抽屉。账本为空 → 连续性抽屉 → 检查记录开关。卡片标题上的状态标签可以帮助快速定位问题所在。
